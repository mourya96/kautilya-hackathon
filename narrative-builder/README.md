# Narrative Builder

Builds dynamic narratives from a large news dataset for any topic. Retrieval is semantic
(FAISS over sentence-transformer embeddings); generation is **RAG with a local LLM served by
vLLM**, orchestrated as a **LangGraph** corrective pipeline; and **RAGAS** measures
hallucination/grounding quality.

## Pipeline

```
load + filter ──▶ embed + FAISS index ──▶ RAG graph ──▶ narrative JSON
                                          (LangGraph)
   retrieve ──▶ generate ──▶ validate ──(fabricated citations)──▶ generate (retry)
                                 │
                                 └──(grounded / out of attempts)──▶ done
```

- **Summary + timeline "why it matters"** are generated by the vLLM model, grounded strictly in
  retrieved articles and cited as `[n]`. A programmatic guard rejects out-of-range citations and
  fabricated article ids and feeds corrective feedback back into generation (up to
  `MAX_GENERATION_ATTEMPTS`), dropping anything still invalid on the final pass.
- **Clusters + relationship graph** remain heuristic (unchanged).
- If vLLM is unreachable or `--no-llm` is set, it falls back to the fully heuristic narrative.

## Setup

```bash
uv python pin 3.12      # the RAG/eval stack has no 3.14 wheels
uv sync
```

## Run a local model with vLLM

```bash
vllm serve Qwen/Qwen2.5-7B-Instruct --port 8000
```

Point the app at it in `.env` (`VLLM_BASE_URL`, `VLLM_MODEL`, …). The judge LLM for RAGAS reuses
the same endpoint by default (override with `RAGAS_JUDGE_*`); RAGAS embeddings reuse
`EMBEDDING_MODEL`.

## Generate a narrative

```bash
uv run python narrative_builder.py --topic "Hyderabad Metro Phase 2B" --top-k 20
uv run python narrative_builder.py --topic "..." --no-llm        # heuristic only
uv run python narrative_builder.py --config                      # show effective config
```

Output JSON includes `generation_mode` (`rag_vllm` or `heuristic`), `narrative_summary`,
`timeline`, `clusters`, and `graph`.

## Evaluate hallucination (RAGAS)

Golden set lives in `eval/golden_set.jsonl` (`{"topic": ..., "reference": ...}` per line; edit it
to match your data). Runs faithfulness, answer relevancy, context precision and context recall:

```bash
uv run python narrative_builder.py --evaluate --eval-set eval/golden_set.jsonl
# or standalone:
uv run python evaluation.py
```

Scores (per-query + aggregate means) are printed and written to `eval_results.json`.
